# Group By Cell

## Overview

Group By cells aggregate data by dimensions. They are used for:
- Aggregating metrics (SUM, COUNT, AVG, MIN, MAX)
- Rolling up data by specific dimensions
- Computing totals and subtotals
- Deduplication with COUNT DISTINCT

## Content Structure

Content — uses `column_name` (NOT `name`), has NO `sql` field:

```json
{
  "table_name": "cell_2_sql_query_label_1140862697",
  "group_by_columns": ["date", "campaign_name", "advertising_channel_type"],
  "aggregations": [
    {
      "column": {
        "column_name": "spend",
        "type": "number"
      },
      "aggregation_type": "sum"
    }
  ]
}
```

## Key Rules

- `table_name`: source table (often a virtual cell output)
- `group_by_columns`: array of column name strings
- Each aggregation: `{"column": {"column_name": "...", "type": "..."}, "aggregation_type": "..."}`
- **`column_name` NOT `name`** — this is a common mistake
- **There is NO `sql` field** — SQL is auto-generated by the backend
- Aggregation types: `sum`, `count`, `count_distinct`, `avg`, `min`, `max`
- Context for run: `"group_by"`

## CRITICAL: Use column_name, NOT name

**WRONG:**
```json
{
  "column": {
    "name": "spend",  // ❌ WRONG
    "type": "number"
  },
  "aggregation_type": "sum"
}
```

**CORRECT:**
```json
{
  "column": {
    "column_name": "spend",  // ✅ CORRECT
    "type": "number"
  },
  "aggregation_type": "sum"
}
```

## Aggregation Types

### sum
Sum all values in the column:
```json
{
  "column": {"column_name": "spend", "type": "number"},
  "aggregation_type": "sum"
}
```

### count
Count total number of rows (including duplicates):
```json
{
  "column": {"column_name": "campaign_id", "type": "string"},
  "aggregation_type": "count"
}
```

### count_distinct
Count unique values (deduplication):
```json
{
  "column": {"column_name": "campaign_id", "type": "string"},
  "aggregation_type": "count_distinct"
}
```

### avg
Average of values:
```json
{
  "column": {"column_name": "ctr", "type": "number"},
  "aggregation_type": "avg"
}
```

### min
Minimum value:
```json
{
  "column": {"column_name": "date", "type": "date"},
  "aggregation_type": "min"
}
```

### max
Maximum value:
```json
{
  "column": {"column_name": "date", "type": "date"},
  "aggregation_type": "max"
}
```

## Multiple Aggregations

You can aggregate multiple columns at once:

```json
{
  "table_name": "cell_1_sql_query_label_123456",
  "group_by_columns": ["date", "campaign_name"],
  "aggregations": [
    {
      "column": {"column_name": "spend", "type": "number"},
      "aggregation_type": "sum"
    },
    {
      "column": {"column_name": "clicks", "type": "number"},
      "aggregation_type": "sum"
    },
    {
      "column": {"column_name": "impressions", "type": "number"},
      "aggregation_type": "sum"
    },
    {
      "column": {"column_name": "conversions", "type": "number"},
      "aggregation_type": "sum"
    }
  ]
}
```

## Empty Template

```json
{
  "cell_type": "group_by",

    "table_name": "",
    "group_by_columns": [],
    "aggregations": []
  },
  "title": "Group by",
  "context": "group_by"

```

## Full Example: Daily Campaign Aggregation

Aggregating ad-level data to daily campaign totals:

```json
{
  "table_name": "cell_2_sql_query_label_1140862697",
  "group_by_columns": [
    "date",
    "campaign_id",
    "campaign_name",
    "advertising_channel_type",
    "__account_id"
  ],
  "aggregations": [
    {
      "column": {
        "column_name": "spend",
        "type": "number"
      },
      "aggregation_type": "sum"
    },
    {
      "column": {
        "column_name": "clicks",
        "type": "number"
      },
      "aggregation_type": "sum"
    },
    {
      "column": {
        "column_name": "impressions",
        "type": "number"
      },
      "aggregation_type": "sum"
    },
    {
      "column": {
        "column_name": "conversions",
        "type": "number"

    }
  ]

```

## Common Use Cases

### 1. Daily Campaign Totals
Roll up ad/adset metrics to campaign level by date:
```
GROUP BY date, campaign_id, campaign_name
AGGREGATE SUM(spend), SUM(clicks), SUM(impressions)
```

### 2. Account Summary
Get totals across all campaigns per account:
```
GROUP BY __account_id, account_name
AGGREGATE SUM(spend), COUNT_DISTINCT(campaign_id)
```

### 3. Deduplication
Remove duplicate rows by keeping unique combinations:
```
GROUP BY unique_key_1, unique_key_2
AGGREGATE MAX(last_updated_date)
```

### 4. Time Period Aggregation
Roll up to weekly or monthly:
```
GROUP BY week_start_date, campaign_id
AGGREGATE SUM(spend), AVG(ctr)
```

Note: For complex date calculations, first add a formula in Blend Data to compute week/month, then group by that column.

## Performance Tips

- Include all key dimensions in `group_by_columns` that you'll need downstream
- Don't over-aggregate early — keep granularity you might need later
- Use `count_distinct` sparingly on high-cardinality columns (slow)
- Always include `__account_id` if you want clickable links in dashboards
